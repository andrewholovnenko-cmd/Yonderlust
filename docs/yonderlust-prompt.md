# Yonderlust — промпт для билда фронтенда

> AI-турагентство: пользователь даёт бюджет + свободные дни, продукт думает за него, куда поехать, и собирает поездку. Не агрегатор дешёвых билетов — концирж.
>
> **Поправка (стек): Next.js 15 (App Router) вместо Vite.** Роутинг file-based в `src/app/**` (`page.tsx`), без React Router. Env с префиксом `NEXT_PUBLIC_`. Картинки через `next/image`. Auth через `@supabase/ssr`. Остальное по спеке.

---

## 0. Контекст и допущения (поправь, если не так)

Фиксирую решения, которые в исходнике были не определены. Если что-то не совпадает — скажи до старта:

- **Только веб**: Next.js 15 (App Router) + React 19 + TypeScript (strict). Не Expo.
- **Frontend-first.** Бэкенд делает друг. Поэтому ключевое архитектурное решение — **контракт-first**: мы прямо сейчас определяем TS-типы (`services/types.ts`) и интерфейс сервисного слоя, под который позже встанет его API. Компоненты не знают, откуда приходят данные.
- **AI-планировщик без бэкенда реально работать не может.** Client-side вызов LLM = слитый API-ключ — так нельзя. Планировщик отдаёт **мок-итинерарии** через тот же сервисный слой с реалистичной задержкой. Когда у друга появится endpoint — меняется одна реализация (`mock → api`), UI не трогается.
- **Supabase сейчас — только auth** (Google OAuth). Как data-layer подключается позже.

## 1. Что строим

**Yonderlust** — AI-консьерж для путешествий.
Вход: бюджет + диапазон свободных дат (+ город вылета + «вайб»/предпочтения).
Выход: ранжированные идеи поездок с перелётом, жильём, активностями и итоговой стоимостью — которые можно сохранить и собрать.

Два режима:

| Режим | Роль | Суть |
|---|---|---|
| **Discover** (AI-планировщик) | **Главная фича, hero** | Юзер даёт ограничения + вкус → получает курированные идеи куда поехать |
| **Plan** (ручной) | Вторичный | Юзер уже знает направление → собирает перелёт + жильё + активности в итинерарий под бюджет |

При первом заходе с лендинга должно быть **мгновенно понятно**, что это что-то вроде премиум-турагентства, а не SaaS-дашборд.

## 2. Стек (зафиксирован)

- **Next.js 15 (App Router) + React 19 + TypeScript** (strict, без `any`)
- **Роутинг — Next.js App Router** (file-based, `src/app/**`), без React Router
- **Tailwind CSS** + слой дизайн-токенов на CSS-переменных (`styles/tokens.css`)
- **Framer Motion** — анимации (по принципам Emil Kowalski)
- **TanStack Query** — фетчинг/кэш (чтобы свап mock→real был чистым)
- **Zod** — рантайм-валидация будущих ответов API
- **@supabase/supabase-js** — auth сейчас, data позже
- **lucide-react** — UI-иконки (line icons). Иллюстрации/бренд/лоадер — кастомный inline SVG
- Стейт: минимум. TanStack Query + при необходимости немного Zustand

## 3. Архитектура: контракт-first + swappable data layer (самое важное)

Весь доступ к данным идёт через **интерфейс сервисов**. Под интерфейс — две реализации: `mock` (типизированные фикстуры с искусственной задержкой) и `api` (заглушки / env-gated endpoints друга). Переключение через `NEXT_PUBLIC_DATA_SOURCE=mock|live`.

```
src/
  app/                 # роутер, провайдеры, layout
  pages/               # Landing, Discover, TripDetail, Plan, Saved
  features/
    planner/           # AI-режим: шаги ввода → генерация → борд результатов
    manual/            # ручной режим: сборка итинерария
    auth/              # AuthProvider, useAuth, ProtectedRoute
  components/          # переиспользуемый UI (Button, Card, Skeleton…)
  components/loaders/  # фирменный лоадер (планета + самолёт)
  services/
    types.ts           # ← КОНТРАКТ с бэкендом друга (единый источник правды)
    tripService.ts     # interface + mock + api
    authService.ts
    index.ts           # выбор реализации по NEXT_PUBLIC_DATA_SOURCE
  lib/
    supabase.ts
    queryClient.ts
  mocks/               # фикстуры: TripIdea, destinations, flights…
  styles/
    tokens.css         # дизайн-токены
  hooks/
```

> Принцип: бэкенд приходит позже → меняем **реализацию**, а не компоненты. Все ответы прогоняем через Zod-схемы, повторяющие типы выше.

## 4. Экраны / роуты

У каждого экрана обязательны три состояния: **loading (skeleton)**, **empty**, **error**.

- **`/` Landing** — сразу читается как «AI-турагентство». Hero с value prop, фирменный лоадер как визуальный мотив, заметный CTA **«Plan my trip»**, ведущий прямо в Discover. Полоса sample-направлений (живая, из моков). CTA на вход.
- **`/discover` (AI-планировщик)** — пошаговый ввод (город вылета → бюджет → даты с тогглом «гибкие» → кол-во людей → выбор вайбов) → **состояние генерации** (фирменный лоадер «AI подбирает поездки») → **борд результатов**: ранжированные карточки `TripIdea` с match-reasons и разбивкой стоимости, с фильтрами. Клик → деталь.
- **`/trip/:id`** — полная идея: итинерарий по дням, перелёт + жильё + активности, итоговая стоимость, «Сохранить» / «Собрать поездку». Переход список↔деталь через shared-layout анимацию.
- **`/plan` (ручной)** — выбор направления + даты → сборка перелёта/жилья/активностей в итинерарий, с **бюджет-баром**, который анимированно заполняется и краснеет при превышении.
- **`/saved`** — сохранённые поездки (**требует auth**).
- **`/auth/callback`** — обработка OAuth-редиректа.

Просмотр **не** гейтим за авторизацией. Гейтим только сохранение.

## 5. Auth (Supabase + Google OAuth)

- `supabase.auth.signInWithOAuth({ provider: 'google' })`
- `AuthProvider` + `useAuth()` хук + `ProtectedRoute` обёртка + sign-out + `/auth/callback`.
- Env: `NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY`. Redirect URLs настроены в проекте Supabase.
- Сессия персистится; UI реагирует на смену состояния (аватар/имя в хедере, кнопка входа/выхода).

## 6. Дизайн

**Используй установленные скиллы (ui/ux pro, Emil Kowalski) — но в рамках этого направления:**

- **Позиционирование:** премиум, editorial, «quiet luxury» travel — высокий концирж, не бюджетный агрегатор. Уверенный whitespace, крупная типографика, сдержанная палитра, фото на первом плане. **Никаких generic SaaS purple-blue градиентов.**
- **Палитра (CSS-токены):** база off-white/ivory + глубокий чернильный почти-чёрный + **ОДИН** характерный акцент. Возьми **deep teal**. Всё в `tokens.css`.
- **Типографика:** характерный display-serif для заголовков (editorial) + чистый grotesk/sans для UI (Inter / Geist). Заданная type-scale.
- **Spacing / radius / shadow:** единая шкала токенов; крупные мягкие радиусы; деликатные тени.
- **Иконки:** lucide-react (line). Иллюстрации/бренд/empty-states/лоадер — кастомный inline SVG.
- **Фото:** качественная destination-фотография (Unsplash-style placeholder), фиксированные aspect-ratio контейнеры, blur-up.

### Эмодзи — строго запрещены

Никаких эмодзи **нигде**: UI, тексты, плейсхолдеры, alt-тексты, моки, комментарии в копирайте. Только SVG/lucide.

## 7. Анимации

**Принципы Emil Kowalski:** осмысленно, быстро, с правильным easing (кастомный cubic-bezier, ~150–300ms), spring для интерактивных элементов, анимируем `transform`/`opacity` (не layout), уважаем `prefers-reduced-motion`. Никакого движения ради движения. Единый набор токенов длительности/easing.

**Фирменный лоадер:** самолёт облетает планету, контрейл формирует светящееся 3D-кольцо. SVG + Framer Motion (орбита через rotation группы, кольцо через анимацию `stroke-dashoffset`). Используется на initial load и как состояние «AI генерирует поездки».

**Набор:** page transitions, lift карточек на hover, staggered появление карточек, заполнение бюджет-бара, переходы между шагами планировщика, skeleton shimmer, shared-layout список↔деталь.

## 8. Мок-данные

~8–12 реалистичных `TripIdea` под разные бюджеты и направления — с разбивкой стоимости и match-reasons, чтобы UI выглядел живым. Уклон под Вену как город вылета: **Montenegro (Kotor), Greece (Naxos/Athens), Malta, Madeira, Lisbon, Split, Tirana** и т.п. Искусственная задержка сервисов 600–1200ms, чтобы отрабатывали loading-состояния. Без эмодзи в текстах моков.

## 9. Качество (non-functional)

- TS strict, без `any`. ESLint + Prettier, ноль ошибок.
- **Responsive**, mobile-first — планировщик и карточки обязаны работать на телефоне.
- **A11y:** семантический HTML, focus-states, клавиатурная навигация, alt-тексты, контраст, reduced-motion.
- **Perf:** lazy-load роутов, lazy images, без layout thrash.
- `.env.example` со всеми переменными. README: как запустить + **как свапнуть mock→бэкенд**.
